Java Method Overriding And Methods Hiding

Method Overriding

Method Overriding :
Changing the implementation of inherit super class method in sub class is known as method overriding .
or
Providing the implementation of super class method signature in sub class is known as method overriding .

Rules of method overriding

Rules of method overriding is :
  • Inheritance is must be required (It is only be written in sub class ).
  • Method name ,argument and return type must be same as super class method .
  • Access modifier can be change ,but weaker access privilege can not assigned with respect to super class method .
    Access modifier lower to higher :
    private -> default -> protected -> public .
  • private method of super class can not overriding
  • final method of super class can not overriding
  • A sub class in a difference package can only overriding the non-final and declared public and protected method .

Example of Method Overriding

Example : 

class A{
    void m1(){
        System.out.println("m1() method in A class");
    }
    void m2(){
        System.out.println("m2() method in A class");
    }
}
class B extends A {
    void m1(){
        System.out.println("m1() method in B class");
    }
}
public class OverridingTest {
    public static void main(String[] args)
    {
        B b = new B();
        b.m1();     // m1() method in B class // class A m1() method is Override by class B m1() method.
        b.m2();     //  m2() method in A class
    }
}

Example of Method Overriding with super class reference

Example : 

class Person{
    public void isEducted (){
        System.out.println("Person can be educated or not");
    }

    public void isMove (){
        System.out.println("Person can move");
    }
}
class Student extends Person{

    @Override
    public void isEducted (){
        System.out.println("Students are educated");
    }
}
public class OverrideTest1 {
    public static void main(String[] args) {
        Person person = new Student();
        person.isEducted();  // isEducted() method is execute in Student class
        person.isMove();    // isMove() method is execute in Person class

    }
}

Can we Override java main() method ?

No ,we can not overload java main() method.

Can we override static method ?

No, static method cannot be overridden. Because static method is bound with class , If we can try to do then then it is called Hiding .

Can we override private methods ?

No, private method cannot be overridden. They are not at all inherited to sub class.

Methods Hiding

Methods Hiding : Creating static method in sub class with same signature as super class is known as method Hiding.

Super class & sub class in Methods Overriding and Hiding

Super Class Sub Class
Non-static method Non-static method same signature is known as Methods Overriding
Static method Static method same signature is known as Methods Hiding
Static method Non-static method same signature , Compile time error
Non-static method Static method same signature , Compile time error

Difference between Overloading and Overriding ?

Method Overloading Method Overriding
Overloaded method have same name but different argument Overriding method have same name same argument with respect to super class method.
Overloaded method have any return type . Overriding method have same return type .
Method overloading is used to increase the readability of the program. Method overriding is used to provide the specific implementation of the method that is already provided by its super class.
Inheritance is not required for method Overloading . Inheritance is not mandatory for method Overriding (IS-A relationship) .
Overloading achieve in either same class or different class . Overriding is situated in sub class .
Private static and final method can be Overloaded Private static and final method can not be Overriding
Method overloading is the example of compile time polymorphism. Method overriding is the example of run time polymorphism.
Constructor can be Overloaded Constructor can not be Overriding

Overview Of JSP

JSP : JSP is Java Server Pages is a server side technology for developing dynamic web pages.
JSP called high level abstraction of Servlet .
  • Java Server Pages is a java based web technology
  • Jsp can also be used to access JavaBeans objects.
  • Jsp ia J2EE technology, For specification for web container manufacturers.
  • JSP is an API .
  • JSP is a dynamic web resource of a web application.
  • JSP is web server side piece of code that extends functionality of web server.
  • Jsp engine is a software written in java technology according to JSP specification.
  • JSP does not require additional files like, java class files, web.xml
  • Any change in the JSP code is handled by Web Container(Application server like tomcat), and doesn't require re-compilation.
  • JSP can be used for separation of the view layer with the business logic in the web application.
  • It is much better than Common Gateway Interface (CGI).

Advantage of JSP

Advantage of JSP :
  • Easy to maintain and code.
  • JSP technology is the extension to servlet technology. We can use all the features of servlet in JSP.
  • High Performance and Scalability.
  • JSP is built on Java technology, so it is platform independent.
  • JSP is enables to separate presentation layer with the business logic layer in the web application.
  • JSP can also include the database connections into it.

Life Cycle of JSP

Life Cycle of JSP : A JSP page is converted into Servlet in order to service requests. The translation of a JSP page to a Servlet is called Lifecycle of JSP.
  • Translation Phase
  • Instantiation Phase / Object Creation
  • Initialization Phase
  • Service Phase
  • Destroy phase

Life Cycle method of JSP :
  • jspInit() (Can be Override )
  • _jspService(HttpServletRequest req, HttpServletResponse resp) (Can not Override )
  • jspDestroy() (Can be Override )
Note : Most contain of JSP file into the _jspService() method .

Java Servlet

Server side programming

Web server : It is a software which stores and manage the resources .

Deployment : Making the resources available to the server is known as deployment .
The resources of any kind are always deploy on to the server .
There are two different type of deployment which are
Manual Deployment : In case manually deployment all the resources are made available to the server by manually .
Automated Deployment : In case automated deployment all resources are deploy no to the server with the help of automated tool such as ANT, MAVEN etc.

Whenever we have to deploy a web application onto server we deploy it in the form of 'war' file , that is the compress version of web application .

Note : In case of Apache Tomcat we deploy all the web application on to a folder of called webapps , present within the Apache Tomcat server .
we can deploy either a web application or created war file and deploy onto webapps folder of application Apache Tomcat server .

Note :
  • Any server is basically used to provide a run time environment .
  • A web server can run only web application .
  • An application server can run both web application and enterprise application .

Apache Tomcat server

Pre-required for Apache Tomcat server : There are three different pre-required for Apache Tomcat server
  • JAVA_HOME
  • CATALINA_HOME
  • PATH
  • JRE_HOME (optional)
JAVA_HOME : This is the path contain all the related folder with respect to JDK which is a part of java folder present in Program File of C drive .

CATALINA_HOME : This is the path contain all the related folder with respect to Apache Tomcat server present in respective drive or directory .

PATH : This is the part of Environment variable which contains the enter path of Apache Tomcat server ,including with bin folder .

JRE_HOME: This is the path contain all the related folder with respect to JRE which is a part of java folder present in Program File of C drive .

The different folder of Apache Tomcat server :
     1 ) bin ,      2 ) conf ,      3 ) lib ,      4 ) logs ,      5 ) webapps ,      6 ) work

bin : bin folder contains a startup batch file and a shutdown batch file which is use to start and stop the server .

conf : conf folder contains a set of configuration files with Apache Tomcat server .

lib : lib folder contains a set of library in the from of jar file .

logs : logs folder contains log message which display on server console , since server console has limited memory .

webapps : webapps folder is use to peploy web application onto the Apache Tomcat server .

work : work folder contains the data with respect to translated server .

Servlet

Servlet : Servlet is a server side program which can perform all the three default type of logic , process the http client request and give http response .

Note :
All the application in a server is manage by JEE Container .
All the servlet with in an application is manage by Servlet Container .

Types of Servlet :
There are two different type of servlet which are as flow

  • Generic servlets
  • HTTP servlets

Generic servlets

Generic servlet :

Generic servlet is not specific to any protocol .
Generic servlet does not support session .
GenericServlet is an abstract class which contain abstract method called service() need to overriding mandatory , science it is abstract method .
Whenever we try to override the service() method it throw an exception ServletException and IOException .
GenericServlet contain two different concrete method where overriding this method are optional as flow
  • public void init(ServletConfig config)
  • public void destroy()
GenericServlet as present in the package call javax.servlet;

Syntax : 

import java.io.IOException;
import javax.servlet.ServletException;
import javax.servlet.ServletRequest;
import javax.servlet.ServletResponse;

public class MyGenericServlet extends javax.servlet.GenericServlet{

    @Override
    public void service(ServletRequest req, ServletResponse resp) throws ServletException, IOException {

        //Write all the implementation logic

    }
}

Http Servlet

Http Servlet :

Http Servlet is specific to particular type of protocol http protocol only .
Http Servlet dose supported session .
HttpServlet is abstract class which is present in javax.servlet.http package .
HttpServlet dose not contain any abstract method .
HttpServlet give appropriate doXxx() method for every type of http request .
Science HttpServlet contain only concrete method where need to override suitable doXxx() method such as
      1 ) doGet ,      2 ) doPost ,       3 ) doPut ,      4 ) doDelete ,       5 ) doOptions ,      6 ) doHead

Example :

protected void doGet(HttpServletRequest req, HttpServletResponse resp)

Syntax : 

import java.io.IOException;
import javax.servlet.ServletException;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

public class MyHttpServlet extends HttpServlet{

    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {

        //Write all the implementation logic

    }
}

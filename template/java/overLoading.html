Method OverLoading and Method Hiding

Method OverLoading

Method OverLoading : Creating multiple method with same name and different argument is known as Method OverLoading.

  • Argument are should be different only when there is different either with number of Argument or Argument type .
  • When overload method is called java involve method base on arrguments.
  • Method overloading increases the readability of the program.
  • You can not define more than one method with the same name, Order and the type of the arguments. It would be compiler error.

Different ways of doing overloading methods

There are three ways of doing overloading methods :

  • The data types of the parameters of methods.
  • The number of parameters in two methods.
  • The Order of the parameters of methods.


The number of parameters in two methods : 
class Box{
    public void area(int a){
        System.out.println( "Area of square is - "+a * a);
    }
    public void area(int a, int b){
        System.out.println( "Area of rectangle is - "+ a * b);
    }
}

public class AreaTest{
     public static void main(String[] args) {
         Box box = new Box();
         box.area(10);
         box.area(10,15);
     }
 }


The data types of the parameters of methods : 
class Addition{
    public void add(int a, int b){
        System.out.println("integer addition - "+ (a+b) );
    }
    public void add(double a, double b){
        System.out.println("double addition - "+ (a+b) );
    }
}

public class AdditionTest{
     public static void main(String[] args) {
         Addition addition = new Addition();
         addition.add(10,15);
         addition.add(10.00,15.00);
     }
 }


The Order of the parameters of methods : 
class Student {
    public void display(String name, int id){
         System.out.println( "The student name - "+ name +"  Id - "+id);
    }
    public void display(int id, String name){
         System.out.println( "The student name - "+ name +"  Id - "+id);
    }
}

public class StudentTest{
     public static void main(String[] args) {
         Student student = new Student();
         student.display("Akash",10);
         student.display(10,"kamal");
     }
 }

Can we overload java main() method?

yes ,we can overload java main() method. But JVM calls main() method which receives string array as arguments only.

Example : 
public class TestMainMethodOverloading{

    public static void main(String[] args){
        System.out.println("main with String[]");
    }
    public static void main(String args){
        System.out.println("main with String");
    }
    public static void main(){
        System.out.println("main without args");
    }
}

Can we overload changing the return type ?

No, method overloading is not possible by changing the return type of the method only because of ambiguity . If try to write then you will get Compile tome error .

Can we declare overloaded methods as final?

Yes, we can declare overloaded methods as final.

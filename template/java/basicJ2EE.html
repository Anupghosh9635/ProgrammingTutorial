Java Enterprise Edition

Java Enterprise Edition (JEE) It is a specification for developing web and enterprise application given in the from of abstraction API to achieved lose coupling between application and server .

Few of JEE API :
     1 ) Servlet ,       2 ) JSP (Java Server Page) ,           3 ) EJB (Enterprise Java Bean) ,      4 ) JMA (Java Mail API) ,      5 ) JTA (java Transaction API ) ,      6 ) JPA (Java Persistence API)

Note : Since server is basically provide the runtime environment , we have to always deploy all the resources on to the server mandatorily in the form of class file .

Software Application

Software Application : A set of code which is use to perform a specific task is known as Software Application .

Contain of software application :
  • User Interface
  • Business Logic
  • Data Base
User Interface : It is media which help in comminuting the user to any device .

Business Logic : It is the logic which helps in getting connecting to particular application.

Data Base : Data base is used to deal with data only.

Type of Software Application

There are three type of Software Application .
  • Stand alone
  • Clint / Server
  • Web Application
Stand alone : Computer software that can work offline, i.e. does not necessarily require network connection to function.

Clint / Server : A client/server application is a piece of software that runs on a client computer and makes requests to a remote server.

Web Application : Applications of this type typically support connected scenarios and can support different browsers running on a range of operating systems and platforms.

Design patterns

An optimize solution for commonly recurring design problem or design need is known as design patterns .

Categorize of design patterns :

  • Creational design pattern
  • Factory design pattern

Factory design pattern

Factory design pattern : Factory is the one which create or produces different object of same or similar type .

Factory generally takes an input of String type .

Note : All the implementation object are created by using facory method or helper method with in a factory class . 

class Car{
}
class Audi extends Car{
}
class Benz extends Car{
}
public class CarFactory {
    public static Car getCar(String type){
        if(type.equalsIgnoreCase("Audi")){
            return new Audi();
        }else if(type.equalsIgnoreCase("Benz")){
            return new Benz();
        }else
        return null;
    }
}

Abstraction

Abstraction : Hiding the implementation and providing the functionality to the user is known as Abstraction .

Output of Abstraction : Output of Abstraction is Loose Coupling .

Loose Coupling : Change the implementation which does not effect the user is known as loose Coupling .
We can achieved Loose Coupling by the help of interface.

interface : It is media type which helps in communicating the user and any device .

Diffrent logic of Factory design pattern

Factory design pattern deals with three (3) type of logic, this are
  • Implementation logic
  • Object creating logic
  • Consumer / utilization logic



Example : 

public interface ISwitch{
    void switchOn();
    void switchOff();
}

public class TubeLight implements ISwitch{
    // Implementation logic
    @Override
    public void switchOn() {
        System.out.println("Tube Light is on ....");
    }
    @Override
    public void switchOff() {
        System.out.println("Tube Light is off ....");
    }
}
public class LedLight implements ISwitch{
    // Implementation logic
    @Override
    public void switchOn() {
        System.out.println("LED Light is on ....");
    }
    @Override
    public void switchOff() {
        System.out.println("LED Light is off ....");
    }
}

public class LightFactor{
    // Object creating logic
    public static ISwitch getlight(String type){
        if(type.equalsIgnoreCase("TubeLight")){
            return new TubeLight();
        }else if(type.equalsIgnoreCase("LedLight")){
            return new LedLight();
        }else{
            System.out.println("No implementation found ");
        }
        return null;
    }
}

public class LightFactorTest {
    // Consumer logic
    public static void main(String[] args) {
        java.util.Scanner scn = new java.util.Scanner(System.in);
        System.out.println("Enter the type of light ");
        String type = scn.next();
        scn.close();
        ISwitch iSwitch = LightFactor.getlight(type);
        if(iSwitch != null){
            iSwitch.switchOn();
            iSwitch.switchOff();
        }
    }
}

Class Loading

Loading a class file in the JVM memory is known as Class Loading .

There are two different way to load a class file .

1 ) .   By calling any of the member of class such as method , constructor , block etc.

2 ) .   By using static method forName() .
Which present in the java.lang package .
This method may possible throw checked Exception call ClassNotFoundException handle it by using try and catch statement .

Example : 

package com.anup.test;
class Test {
    static {
	System.out.println("Test class is loading .... ");
    }
}
public class ClassloadingTest {
    public static void main(String[] args) {
	try {
	    Class.forName("com.anup.test.Test");
	} catch (ClassNotFoundException e) {
	    e.printStackTrace();
	}
    }
}

Costly resource / heavy weight object

Costly resource / heavy weight object :

  • Resource which makes use of sysytem properties or network properties in the from of system is know as Costly resource or heavy weight object .
  • Costly resource must be close close in general since it effect the performance of the application .
  • It is always a good practices to close all costly resource with in finally block , using if condition to avoid NullPointerException .

Example : 

public class Test {
    public static void main(String[] args) {
	FileReader fr =null;
	try {
	    fr = new FileReader("data.text");
	} catch (FileNotFoundException e) {
	    e.printStackTrace();
	}finally {
	    if (fr != null){
		try {
		    fr.close();
		} catch (IOException e) {
		    e.printStackTrace();
		}
	    }
	}
    }
}

What is exception handling ?

The process of converting system error messages into user friendly error message is known as Exception handling.

Hierarchy of Java Exception classes

Exception propagation

Exception propagation : passing the Exception from one method to another method is known as Exception propagation. 

public class ExceptionPropagation {
    void method3() {
        int result = 10 / 0; // Exception Generated
    }
    void method2() {
        method3();
    }
    void method1() {
        try {
            method2();
        } catch (Exception e) {
            System.out.println("Exception is handled here");
        }
    }

    public static void main(String args[]) {
        ExceptionPropagation obj = new ExceptionPropagation();
        obj.method1();
    }
}

Types of exceptions

There are two types of exceptions
  • Checked exceptions
  • Unchecked exceptions

Checked exceptions

Checked exceptions : Exception Which are directly subclass of the Exception class is known as Checked exceptions.
or All exceptions other than Runtime Exceptions are known as Checked exceptions.
  • Checked exceptions are checked in compile time .
  • try catch is mandatory for checked exceptions other wise compiler generate compile time error.

Unchecked exceptions

Unchecked exceptions : Exception Which are subclass of the RunTimeException are known as Unchecked exceptions.
  • Unchecked exceptions are not checked in compile time .
  • try catch is not mandatory for Unchecked exceptions .

try , catch and finally blocks

try : The try block contains set of code which might occur exception (or risky statement).
  • try block must be used within the method.
  • try block must be followed by either catch or finally block.

catch : The catch block is used to handle the exceptions. .
  • A catch block must be associated with a try block.
  • multiple catch blocks are allow for a single try .
  • when there are multiple catch blocks,catch block order should be from specific type to generic type .

finally : finally is execute always where there is exceptions or not .
  • finally block is used to execute important code such as closing database connection , closing file etc.
  • finally is used to along with try or try catch .
  • only one finally keyword is allow for particular try .

Throw and Throws Keyword

throw Keyword : throw keyword is used to throw an exception explicitly.
throw keyword is used inside method body .

throws Keyword : throws keyword is used to declare an exception .
throw throws
throw keyword is used to throw an exception explicitly . throws keyword is used to declare an exception.
throw is used within the method. throws is used with the method signature.
throw is followed by an object . throws is followed by class.
throw can throw only one exceptions. throws can declare multiple exceptions.

Which situation finally block is not execute ?

finally block is not execute in the situation of below condition
  • When System.exit() is executed.
  • When Object of type Error is thrown .

Error

Error : Error is a subclass of the Throwable class .
  • Errors are the critical conditions that occur due to the lack of the system resources .
  • Errors can not be handled by the code of the program .
  • Ex. OutOfMemoryError occurs when JVM runs out of memory or StackOverflowError occurs when stack overflows etc .

Difference between Error and Exception

Error Exception
All errors in java are unchecked type. Exceptions include both checked as well as unchecked type.
An error is caused due to lack of system resources. An exception is caused because of the code.
We can not handle an error by the program code. Exceptions are handled using three keywords "try", "catch", and "throw".
In Java, errors are defined "java.lang.Error" package. In Java, an exceptions are defined in"java.lang.Exception".
Ex. java.lang.StackOverflowError, java.lang.OutOfMemoryError Ex. Checked Exceptions : SQLException, IOException Unchecked Exceptions : ArrayIndexOutOfBoundException, ClassCastException, NullPointerException

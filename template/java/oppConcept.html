Java OOPs Concepts

Java OOPs Concepts : OOPs stands for Object Oriented Programming. The object-oriented paradigm supports four major principles: abstraction, encapsulation, inheritance, and polymorphism.

What Is an Object ?

Object : object is a real world entity which has state and behavior .
Software object which has variable and method variable represent state and method represent behavior.
Example:   A dog has states-color, name, breed as well as behaviors -wagging, barking, eating.

  • An object is an instance of a class.
  • To create an object at the run time a class is require .
  • Non static variable represent state of an object , use non static variable to store information which is specification object
  • static variable represent state of an class, use static variable to store information not a specific to a particular object
  • The reference values (often just references) are pointers to these objects, and a special null reference, which refers to no object

Inheritance in Java

Inheritance : process of acquiring member of property from one class to another class is known as Inheritance.
  • extends is the keyword used to inherit the properties of a class.
  • Inheritance represents the IS-A relationship.
  • The main Purpose of Inheritance is code reusablety.
  • By default Object Class is extends every class.

SubClass extends SuperClass
SubClass : The class which inherits property of other class is known as SubClass.
SuperClass : The class whose property are inherited is known as SubClass.

Example : 

class A {
    int i = 10 ;
}
class B extends A {
    int j = 20 ;
    void display(){
        System.out.println("i = "+ i);
        System.out.println("j = "+ j);
    }
}
public class Test {
    public static void main(String[] args) {
        B b1 =new B();
        b1.display();
    }
}

Polymorphism

Polymorphism : One object or one method call shows different behavior in different context (situation) it known as polymorphism .

There are two types of polymorphism
  • Compile time polymorphism .
  • Runtime polymorphism .

Compile time polymorphism : Call to overloaded method is resolved in compile time based on parameter type is known as Compile time polymorphism .

Runtime polymorphism : Call to overridden method is resolved at run time based on instance type is known as runtime polymorphism .

Example : 

class Bike{
    public void start(){
        System.out.println("Bike is Start ");
    }
}
class Pulsar extends Bike{
    @Override
    public void start(){
        System.out.println("Pulsar is Start ");
    }
}
class BikeRide{
    public void ride(Bike b){
        b.start();
        System.out.println("ride Bike");
    }
    public void ride(Pulsar p){
        p.start();
        System.out.println("ride Pulsar");
    }
}
public class PolymorphismTest {
    public static void main(String[] args) {
        BikeRide bikeRide = new BikeRide();
        Bike bike = new Pulsar();
        bikeRide.ride(bike);
    }
}

Encapsulation

Encapsulation : process of wrapping code and data together into a single unit is known as encapsulation.
or encapsulation is a technique hiding field (data member) and providing public method to access the field is known as encapsulation.
  • public method which which is use to set the private field is known setter method.
  • public method which which is use to set the private field is known setter method.
  • encapsulation is also refer as data hiding .


Example : 

class Employee {
    private String name;
    public void setName(String name) {
        this.name = name;
    }
    public String getName() {
        return name;
    }
}


public class EncapsulationTest {
    public static void main(String[] args) {
        Employee employee = new Employee();
        employee.setName("Anup");
        System.out.println("Employee Name -> "+employee.getName());

    }
}


Advantage of Encapsulation :
  • A class can have full control on its data member.
  • class can make it data member either read or write by providing request setter, getter
  • Encapsulation allows you to change one part of code without affecting other parts of code.
  • Encapsulation in Java makes unit testing easy.

Abstraction

Abstraction : process of hiding complex implementation and providing required functionality to the user is known as Abstraction .

Abstract method : A method with out an implementation is known as abstract method.
  • Abstract method is declared with abstract key word
  • Abstract method is must be end with semicolon (;) .
  • Example : 

    abstract class AbstractClass {
    abstract public void start();
}


Abstract class : A class which has abstract method is known as abstract class .
  • Abstract class is declared with abstract modifier word
  • Abstract class can be instantiated .
  • With out abstract method we can declared as abstract .
  • If class has one abstract method then it is mandatory to declared class abstract, other wise get Compile time error.
  • Abstract class can be content only concrete method.
  • When a class extends abstract class all abstract method must be Override in subclass, otherwise subclass became abstract class .

Invalid combination of abstract modifiers :
  • abstract and final
  • abstract and static
  • abstract and private


Real time Example of Abstraction : If we think in car they are provide staring for moving the direction , but inside of this complex implementation of every company they are hiding by the cover , for control this they are providing staring .

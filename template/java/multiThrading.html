Java MultiThreading

MultiTasking

  • Execute multiple programs concurrently is known as MultiTasking .
  • In the java MultiTasking is achieved by thread .
  • Thread is light weight subprocess .
  • Each Thread forms an independent path of execution .

MultiThreading

Threading : Thread is light weight subprocess.

MultiThreading : Process of executing multiple threads simultaneously is known as MultiThreading .

Default Threading

When java class is executed the JVM create three thread by default, those are .
  • Garbage collector thread .
  • Thread Scheduler .
  • Main thread .

Way of created thread

There are two way to create thread
1. )
  • Create a class which is extends Thread class .
  • Override the run method .
  • Create an Object of sub class and call start() method .
2. )
  • Create a class which is implements Runnable interface .
  • Create a Thread object by initialing Runnable object and call start() method of the thread object .

methods of Thread class

Method Description
public void run() this method is used to perform action for a thread
public void start() starts the execution of the thread.JVM calls the run() method on the thread.
public void sleep(long miliseconds) Causes the currently executing thread to sleep (temporarily cease execution) for the specified number of milliseconds.
public int getPriority() returns the priority of the thread.
public int setPriority(int priority) changes the priority of the thread.
public String getName() returns the name of the thread.
public void setName(String name) changes the name of the thread.
public int getId() returns the id of the thread.
public void join() waits for a thread to die.
public void join(long miliseconds) waits for a thread to die for the specified miliseconds.
public boolean isAlive() tests if the thread is alive.
public Thread currentThread() returns the reference of currently executing thread.
public Thread.State getState() returns the state of the thread.
public void resume() is used to resume the suspended thread(depricated).
public boolean isDaemon() tests if the thread is a daemon thread.
public void setDaemon(boolean b) marks the thread as daemon or user thread.
public void yield() causes the currently executing thread object to temporarily pause and allow other threads to execute.
public void interrupt() interrupts the thread.
public boolean isInterrupted() tests if the thread has been interrupted.
public static boolean interrupted() tests if the current thread has been interrupted.
public void suspend() is used to suspend the thread(depricated).
public void stop() is used to stop the thread(depricated).

Thread Example by extending Thread class 

public class ThreadTest {
    public static void main(String[] args) {
        Thread th1 = new Thread1();
        Thread th2 = new Thread2();
        th1.start();
        th2.start();
    }
}

class Thread1 extends Thread {
    @Override
    public void run(){
        System.out.println("Character Print Thread is Start");
        for (char c = 'a'; c <= 'z'; c++) {
            System.out.println(c);
            try {
                Thread.sleep(500);
            } catch (InterruptedException ex) {
                ex.printStackTrace();
            }
        }
        System.out.println("Character Print Thread is End");
    }
}

class Thread2 extends Thread {
    @Override
    public void run(){
        System.out.println("Number Print Thread is Start");
        for (int i = 0; i < 50; i++) {
            System.out.println(i);

            try {
                Thread.sleep(500);
            } catch (InterruptedException ex) {
                ex.printStackTrace();
            }
        }
        System.out.println("Number Print Thread is End");
    }
}

Why do we need threads ?

  • To enhance parallel processing
  • To increase response to the user
  • To utilize the idle time of the CPU
  • Prioritize your work depending on priority

Daemon Thread

Thread types : there are manly two type of thread ,this are bellow.
  • Daemon Thread
  • Non Daemon Thread


Daemon Thread :
  • Daemon Thread are service provider thread for non daemon thread .
  • To execute daemon thread at least one non daemon thread should be execute .
  • All the daemon thread are terminated immediately when last non daemon thread is terminated in the application
  • In the thread class provide a method setDaemon(boolean b), by default it is non Daemon thread .
  • setDaemon() method should be call before the calling of start() method .
  • Example of non daemon thread is Main thread . Example of daemon thread is 1. Thread Scheduler 2. Garbage collector thread

Thread-safe in java

  • When multiple thread are using to share object of the same time modified made by one thread can be effect to other thread execution .
  • to avoid this conflict object method should be synchronize .
  • When the thread enter in synchronize non static method a lock will be create of an same object other thread cant be used the same thread until the lock is reliesed .
Thread-safe class does not allowed more then one thtead at a same time .
To make object thread-safe method should be synchronize.

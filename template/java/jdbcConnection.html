JDBC and step of JDBC Connection

JDBC : java is given the specifically given of the from of abstraction API to communicate between java application and database server, in a loosely couple mannar or databse independent way .

Step of JDBC Connection
  1. Load and register the driver (class) .
  2. Establish a connection with the database server .
  3. Create Statement or platform .
  4. Execute sql query or statement .
  5. Process the Result set data (optional) .
  6. Close the all costly resources .


Internal of Driver :
Criteria of of Driver Implementation :
All Driver class must have static block in it .
All the driver class must be implement java.sql.Driver interface .
All the Driver class must be register with the helper class of JDBC API DriverManager using static method called registerDriver().


JDBC 1st Step

Load and register the driver (class) :

In this step we have to load and register the Driver class with JDBC API which can be done in different ways .
1 )   Creating an object of driver class and register in with DriverManager by using registerDriver() method as flow
      Driver d = new Driver();
      DriverManager.registerDriver(d);
this is not a good practice since it cause tight coupling between java application and database server.

2 )   Another way is using static method called forName() method .
      Class.forName("com.mysql.jdbc.Driver");

Example JDBC 1st Step : 

public class JDBC1stStep {
    public static void main(String[] args) {
	try {
	    Class.forName("com.mysql.jdbc.Driver");
	    System.out.println("Driver loaded Sucessfully ........");
	} catch (ClassNotFoundException e) {
	    e.printStackTrace();
	}
    }
}



JDBC 2nd Step

Establish a connection with the database server :


We can establish a connection with the database server by using the getConnection() method .
getConnection() method is static factory method method or helper method which is used to establish a connection with the data base server .
getConnection() method is given three(3) different overloaded method as flow
  • public static Connection getConnection(String url);
  • public static Connection getConnection(String url, java.util.Properties info);
  • public static Connection getConnection(String url, String user, String password);
When ever we try to used either of this overloaded method then the method throw a checked Exception called SQLException .

Connection interface :

java.sql.Connection is a interface which is a part of JDBC API and the implementation are provide by different data base server or vender as a part of JDBC Driver .
Since Connection is costly resources ,it has to close within finally block using if condition to avoid NullPointerException ,since the costly resource it effect the performance of an application .

DriverManager :

java.sql.DriverManager is a helper class which is a part of JDBC API and which contain the important static method called
  • public static synchronized void registerDriver(java.sql.Driver driver) ;
  • public static Connection getConnection() (overloaded method)

getConnection() method :

getConnection() method is used to establish a connection with the data base server .
getConnection() method return appropriate implementation object of Connection interface base of url .
Syntax :
Connection con = DriverManager.getConnection( "jdbc:mysql://localhost:3306?user=root&password=password");

Example JDBC 2nd Step : 

import java.sql.*;

JDBC 3rd Step

Create Statement or platform :

Different type of SQL Statement are flows :
  • DML : insert, update, delete
  • DDL : create, alter, drop, rename, truncate
  • DQL : select
To execute any kind of SQL statement we need a statement or platform this platform can be creating by using either Statement, PreparedStatement, CallableStatement
All this are interface is a part of JDBC API and implementation are provide by different data base server or vender as a part of JDBC Driver .
Syntax :
      Statement stmt = con.createStatement();
  • createStatement() method is non-static method of which is declared in Connection interface
  • createStatement() method is a factory method or helper method which is used to execute and return implementation object of Statement interface .


Example JDBC 3rd Step : 

import java.sql.*;
public class JDBC3rdStep {
    public static void main(String[] args) {
	Connection con = null;
	Statement stmt = null;

	    Class.forName("com.mysql.jdbc.Driver");
	    System.out.println("Driver loaded Sucessfully ........");
	    con = DriverManager.getConnection("jdbc:mysql://localhost:3306?user=root&password=password");

	    stmt = con.createStatement();

	}
    }
}


Note :
Whenever we are executed similar reapetade query , Statement interface compile and execute the query on every instance of execution .
    In this case compilation is unnecessary since it is a similarity reapetade query .
Hence Statement is not good choice for execution of similar reapetade query . We can do pass of query at the time of implementation of object creation .

PreparedStatement :

  • java.sql.PreparedStatement interface is a part of JDBC API and implementation are provide by different data base server or vender as a part of JDBC Driver .
  • PreparedStatement is sub interface of Statement interface
  • PreparedStatement is known as preCompile Statement
  • PreparedStatement support the concept of compile once and execute many time (Execution plane) .
  • PreparedStatement support the concept of place holder to pass the value dynamically at the run time .
  • We pass the query at the time of implementation of object creation of PreparedStatement
place holder : place holder is a parameter which hold the value dynamically at the run time .
In JDBC place holder represent as ? .
Syntax :
      String insertQuery = "insert into student.login_info (name, password, mobile_no, sex) values(?,?,?,?)" ;
      String deleteQuery = "delete from student.login_info Where id= ?";
      String updateQuery = "update student.login_info set password=? where id =? ";

Setting the value for place holder :

  • We have to set the data to the place holder before the execution .
  • The number of data must exactly match with the number of place holder .
  • To set the data to the place holder , we have to use setXxx() method
Syntax :
      + void setXxx(int placeHolderNo_Index, Xxx data )
e.g.
      rs.setInt(1,101);
      rs.setInt(2,"Anup");

JDBC 4th Step

Execute sql query or statement :

To execute any kind of sql statement or query , we have three different method
      execute()
      executeUpdate()
      executeQuery()
All this method declared in Statement interface .

execute() method :

execute() method is a generic type of method it can execute all kind of query or statement .
Syntax :
      boolean execute(String sqlQuery);
The return type of execute() method is boolean true or false .
This method return true in the case of DQL statement and return false in case of DQL and DDL statement .
The out come of DML statement 0 to n integer value , which is given number of recode effected we can used the bellow method
      public int getUpdateCount(); // DML statement
The return type of getUpdateCount() methode always integer ,which can execute only DML statement.
Apart from DML ,if either DQL or DDL is executed using getUpdateCount() method , then it throws SQLException .

executeUpdate() method : executeUpdate() method is specialize method which is use to execute only DML statement .
The return type for executeUpdate() is integer . The out come of DML statement 0 to n integer value which is give to number of recode effected .
Apart from DML ,if either DQL or DDL is executed,then it throws SQLException .

JDBC 5th Step

Process the Result set data (optional) :

When ever is execute any DQL query ,then we may get the result which is refer as process data or resulted data .
The process data is store in cursor or buffer memory .
We have to use java.sql.ResultSet to phase the process data from the cursor or buffer memory .
The volume of data in cursor or buffer memory may different from the volume of table data .
Note : Column name known as Column Label .
Column number is known as Column Index .

ResultSet : java.sql.ResultSet is a interface which is a part of JDBC API and the implementation are provide by different data base server or vender as a part of JDBC Driver .
There are two overloaded method of ResultSet which to phase the result data from the courser memory or buffer memory. which is known as getXxx() .
Syntax :
      + Xxx getXxx(int ColumnNumber)   // + means public
      + Xxx getXxx(String ColumnName)
e.g.
          + int getInt("ID")
or       + int getInt(1)

          + String getString("Name")
or       + String getString(2)

+ boolean next() next() method is used only when there are limited number of data present in the database .

+ boolean absolute(int rowNo); next() method is used only when there are n number of recode present in the database .
Note : the return type of getXxx() method is appropriate data type .
Example : 

import java.sql.*;
public class JDBC5thStep {
    public static void main(String[] args) {
        Connection con = null;
        Statement stmt = null;
        String query = " select * from student.login_info ";
        try {
            //  Loading and register the driver
            Class.forName("com.mysql.jdbc.Driver");
            // Establish a connection with the database server
            con = DriverManager.getConnection("jdbc:mysql://localhost:3306?user=root&password=password");
            // Createing Statement or platform
            stmt = con.createStatement();
            // Executing sql query
            ResultSet rs = stmt.executeQuery(query);
           // Processing the Result set data
            while(rs.next()){
                int id=rs.getInt(1);
                System.out.println("id - "+id);
                String name = rs.getString(2);
                System.out.println("name - " +name);
                String mobileNo = rs.getString(3);
                System.out.println("mobileNo - "+mobileNo);
                String password = rs.getString(4);
                System.out.println("password - "+password);
            }
            System.out.println("Execute successfully ...");
        } catch (ClassNotFoundException | SQLException e) {
            e.printStackTrace();
            System.out.println("Execute Faild ...");
        }
    }
}



JDBC 6th Step

Close the all costly resources :

In the JDBC Connection we are use Connection Statement like costly resources ,it has to close within finally block using if condition to avoid NullPointerException , since the costly resource it effect the performance of an application .

Example : 

import java.sql.*;
public class JDBC6thStep {
    public static void main(String[] args) {
        Connection con = null;
        Statement stmt = null;
        String query = " select * from student.login_info ";
        try {
            //  Loading and register the driver
            Class.forName("com.mysql.jdbc.Driver");
            // Establish a connection with the database server
            con = DriverManager.getConnection("jdbc:mysql://localhost:3306?user=root&password=password");
            // Createing Statement or platform
            stmt = con.createStatement();
            // Executing sql query
            ResultSet rs = stmt.executeQuery(query);
           // Processing the Result set data
            while(rs.next()){
                int id=rs.getInt(1);
                System.out.println("id - "+id);
                String name = rs.getString(2);
                System.out.println("name - " +name);
                String mobileNo = rs.getString(3);
                System.out.println("mobileNo - "+mobileNo);
                String password = rs.getString(4);
                System.out.println("password - "+password);
            }
            System.out.println("Execute successfully ...");
        } catch (ClassNotFoundException | SQLException e) {
            e.printStackTrace();
            System.out.println("Execute Faild ...");
        } finally {
            //Close the all costly resources
            if (stmt != null) {
                try {
                    stmt.close();
                } catch (SQLException e) {
                    e.printStackTrace();
                }
            }
            if (con != null) {
                try {
                    con.close();
                } catch (SQLException e) {
                    e.printStackTrace();
                }
            }
        }
    }
}

Batch update

Every individual database call from a java application to database server is consider as costly resources .
The more number of database call reduce the performance of the application ,hence better used Batch update .

Batch update : Batch update is a collection of DML statement which is execute in a single database call.
Batch update generally improved the performance of the application .
Batch update can be executed by using Statement and PreparedStatement interface.

Batch with Statement : There are two important method with respect the
  • addBatch() : This method is use to add multiple DML Statement into a batch .
  • executeBatch() : It is use to execute the enter batch by making a single database call to the database server . The output of the executeBatch() method is an integer array which total number of recode effected. The size of the array gives the total number of DML statement added into batch


Example : 

import java.sql.*;
import java.util.Arrays;
public class BatchUpdateTest {
    public static void main(String[] args) {
        Connection con = null;
        Statement stmt = null;
        String insertQuery = "insert into student.login_info (name, password, mobile_no, sex)"
                      + " values('AnupGh','test','+919635499033','male')";
        String deleteQuery = "delete from admin.member_info Where id='2'";
        String updateQuery = "update student.login_info set password='password' where id ='14'";
        try {
            Class.forName("com.mysql.jdbc.Driver");
            con = DriverManager.getConnection("jdbc:mysql://localhost:3306?user=root&password=password");
            stmt = con.createStatement();
            stmt.addBatch(insertQuery);
            stmt.addBatch(deleteQuery);
            stmt.addBatch(updateQuery);
            int[] arr = stmt.executeBatch();
            System.out.println("print batch Array : "+Arrays.toString(arr));
            System.out.println("Batch update Successfully done.....");
        } catch (ClassNotFoundException | SQLException e) {
            System.out.println("Execute Faild ...");
            e.printStackTrace();
        } finally {
            if (stmt != null) {
                try {
                    stmt.close();
                } catch (SQLException e) {
                    e.printStackTrace();
                }
            }
            if (con != null) {
                try {
                    con.close();
                } catch (SQLException e) {
                    e.printStackTrace();
                }
            }
        }
    }
}


Note :
  • Whenever we use Batch with Statement interface , then a single batch can contain any kind of DML Statement
  • Whenever we use Batch with PreparedStatement interface , then a single batch can contain only one kind of DML Statement

JSBC Transaction

Whenever we establish a connection with database server , all the executed query data are automatically save into the database, since auto commit mode is enable set true by default whenever implicit call .
We can explicit disable auto commit mood by using bellow method .
Syntax : 

 public void setAutoCommit(boolean autoCommit)
e.g. 

  con.setAutoCommit(false);

Once we disable the autoCommit mood none of the data are automatic save into the .
If all the execute query is successful ,then we have to explictly save those data into the database by using the bellow method .
Syntax : 

 public void commit()
e.g. 

  con.commit();

If any one of the query fail then the entire executed database operation must be rollback , where transaction start from the beginning .
JDBC transaction is use to achieved ACID property .

JDBC transaction : JDBC transaction is consider as a single business unite , which may have multiple SQL statement which must be execute .

Example : 

import java.sql.*;
public class JDBCTransactionTest {
    public static void main(String[] args) {
        Connection con = null;
         PreparedStatement pstmt = null;
        PreparedStatement pstmt1 = null;
        String insertQuery = "insert into student.login_info (name, password, mobile_no) values(?,?,?)";
        String insertQuery1 = "insert into student.detail_info (name, add, mobile_no, sex) values(?,?,?,?)";

        try {
            Class.forName("com.mysql.jdbc.Driver");
            con = DriverManager.getConnection("jdbc:mysql://localhost:3306?user=root&password=password");
            pstmt = con.prepareStatement(insertQuery);
            con.setAutoCommit(false);
            // Beginning Transaction
            pstmt.setString(1, "Papun");
            pstmt.setString(2, "password123");
            pstmt.setString(3, "9999999999");
            pstmt.executeUpdate();
            // 1 oprtration done 2nd operation 'll start
            pstmt1 = con.prepareStatement(insertQuery1);
            pstmt1.setString(1, "Papun");
            pstmt1.setString(2, "Kolkata");
            pstmt1.setString(3, "9999999999");
            pstmt1.setString(4, "male");
            pstmt1.executeUpdate();
            // 2nd operation done 
            con.commit();
            System.out.println("Execute successfully .....");

        } catch (ClassNotFoundException | SQLException e) {
            e.printStackTrace();
            System.out.println("Execute Faild ...");
        } finally {
            if (pstmt != null) {
                try {
                    pstmt.close();
                } catch (SQLException e) {
                    e.printStackTrace();
                }
            }
            if (con != null) {
                try {
                    con.close();
                } catch (SQLException e) {
                    e.printStackTrace();
                }
            }
        }
    }
}

Metadata

Metadata :

Data about data is known as Metadata .
Example : DataBase Name , Driver Name , Driver version , no of column present in the DataBase table etc .
Metadata or meta information about DB or Driver or Table can be knowing by using following
  • DatabaseMetaData
  • ResultSetMetaData
Both DatabaseMetaData and ResultSetMetaData are interface which is a part of JDBC API and the implementation provide by different DataBase server or vender .
DatabaseMetaData is use to give meta information about DataBase server or driver name .
ResultSetMetaData is use to give meta information about the table which is associated of ResultSet , which in turn associated with SQL query .

getMetaData() this is helper method which is used to give meta information .
